## Gateway 故障排查记录

### 问题1: Gateway Restart Loop（已解决）

**现象**：Gateway 每 8 秒规律重启，systemd 触发 start-limit-hit

**根本原因**：系统内有两个 Hermes Gateway 进程同时运行，PID 148725 是旧进程，新进程启动时被 Feishu app lock 拦住（code=200340）

**解决步骤**：
1. 找出并 kill 所有残留 gateway 进程（`pgrep -f hermes_cli.main`）
2. 清理 hermes-gateway service 的 zombie 状态
3. 重启：`systemctl --user daemon-reload && systemctl --user reset-failed hermes-gateway && systemctl --user restart hermes-gateway`

**systemd override**（已配置）：
```
[Service]
Restart=always
StartLimitBurst=20
```

**关键教训**：老细通过 systemd restart 而非手动 kill 的方式会导致 zombie 残留进程，下次重启用的是旧 PID，新进程被 Feishu lock 拦死。手动 restart 不会出现这个问题。

### 问题2: Feishu Command Approval Required 200340（已解决）

**现象**：在 Feishu 弹出的 approval card 点任何选项都报错 code: 200340

**根本原因**：同问题1 — Gateway 根本没跑起来（被 Feishu lock 拦住），approval 交互本身失败了

**解决**：清理残留进程后，approval 正常
